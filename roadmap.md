# Roadmap

## Ближайшие направления

- [ ] Сохранение контактов: устойчивые identity устройств, список доверенных друзей, локальные имена, last seen и проверка подлинности при повторной встрече.
- [ ] Прямая связь с контактами: возможность выбрать сохраненный контакт и попытаться подключиться без общей комнаты, если приложение сейчас запущено и nearby-канал доступен.
- [ ] Mesh-сеть: собственный app-level routing поверх nearby-транспортов, включая соседей, `packetId`, TTL, дедупликацию, relay, выбор маршрутов и отдельную voice-политику. Текстовый MESHRA-переключатель для комнат реализовано 2026-07-13; MESHRA voice реализовано 2026-07-14; компактный бинарный MESHRA voice wire-format с 9-байтовым заголовком реализовано 2026-07-19; динамические Nearby topology (`P2P_STAR` для Star, `P2P_CLUSTER` для MESHRA) и чередующийся lobby discovery реализовано 2026-07-19; Direct voice readiness-барьер из рабочего commit `25683d7` восстановлен 2026-07-19; MESHRA link health heartbeat 4 байта с RTT/loss/status реализовано 2026-07-19; UI ping/connect-dot от heartbeat health реализовано 2026-07-19; MESHRA healing discovery останавливается при достаточной active/pending связности реализовано 2026-07-19; heartbeat LOST исключает link из room/event/voice отправки и запускает healing реализовано 2026-07-19; полноценный route selection, jitter/deadline policy и устойчивая реконнект-стратегия еще отдельно.
  - [ ] Приоритеты mesh-рассылки: добавить lightweight рейтинг соседних узлов по их заявленной связности/числу активных link-ов, чтобы event сначала уходил в сторону более связанного gateway. Для MVP достаточно одноуровневой метрики вида `nodeDegree=5`, которую узел публикует соседям; на группах 5–10 устройств это может ускорить распространение без сложного глобального routing-а.
